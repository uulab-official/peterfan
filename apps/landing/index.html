One tool. Every metric.

From per-core load to SMART health, PeterFan reads the sensors your machine already exposes — and presents them cleanly.

CPU

Total & per-core load, live frequency, package temperature.

GPU

Utilization, VRAM usage, and core temperature.

Memory

Used / free, memory pressure, swap, and the top apps eating RAM.

Disk

Free / used capacity, read & write throughput, SMART health.

Network

Up / down speed, total transferred, local & public IP.

Sensors

Temperatures, voltages, power draw, and fan speeds.

Battery

Charge, health, cycle count, and time remaining.

Processes

The top CPU and memory consumers, ranked live.

Fan control

Manual override, a visual curve editor, and profiles: Silent, Balanced, Gaming, Performance, Maximum.

Built once. Native on both.

The same single Rust binary speaks to platform sensor APIs directly — no Electron, no daemons you didn't ask for.

macOS

Apple Silicon & Intel. SMC thermals, fan RPM, battery health, and IOKit sensors.

Windows

WMI & LibreHardwareMonitor-style sensor access for CPU, GPU, drives and fans.

A CLI you'll actually keep open.

Every command is fast, scriptable, and pipeable. Add --json to anything and pipe it into jq, a status bar, or your own dashboard.

  • Human-readable tables by default
  • --json on every subcommand
  • Live TUI dashboard with peterfan dash
  • Local HTTP API planned for Stream Deck, Raycast & Home Assistant
~ zsh 
# live status overview
peterfan status

# machine-readable temps for scripting
peterfan --json temps | jq '.cpu.package'

# switch the fan profile
peterfan profile gaming

# open the live terminal dashboard
peterfan dash

Safe by design.

Touching fan controllers means touching hardware. PeterFan treats that with respect.

Capabilities up front

PeterFan tells you exactly what it will read and write before it does anything privileged.

Read-only first

Monitoring needs no elevated rights. Fan control is strictly opt-in, and never the default.

Restore on exit

Hand control back to firmware automatically when PeterFan quits — your machine is never left stranded.

Get started in one command.

PeterFan ships as a single self-contained binary. Build from source today; package managers are on the way.

Available now

Build with Cargo

~ shell 
# clone & build a release binary
git clone https://github.com/uulab-official/peterfan
cd peterfan
cargo build --release

# run it
./target/release/peterfan status
